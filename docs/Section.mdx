# Section Component

The Section component is a fundamental layout building block that wraps content with consistent vertical padding while allowing for customizable styling and identification.

import Section from '@/components/shared/Section'

interface SectionProps {
  readonly children: React.ReactNode;
  readonly className?: string;
  readonly id?: string;
}

export default function Section({
  children,
  className = "",
  id,
}: SectionProps) {
  return (
    <section id={id} className={`py-20 ${className}`}>
      {children}
    </section>
  );
}

## Props

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| children | React.ReactNode | Yes | - | Content to be rendered inside the section |
| className | string | No | '' | Additional CSS classes to apply |
| id | string | No | undefined | HTML id attribute for the section |

## Basic Examples 

Here's a default section with basic content:

export const DefaultExample = () => (
  <Section>
    <h1>My Content</h1>
    <p>This section has default padding and styling</p>
  </Section>
)

Here's a section with custom styling:

export const StyledExample = () => (
  <Section className="bg-gray-100 text-center">
    <div>Centered section with gray background</div>
  </Section>
)

And a section with an ID for navigation:

export const IdExample = () => (
  <Section id="about">
    <div>This section can be linked to with #about</div>
  </Section>
)

## Usage Guidelines

### Do's
- Use for major page segments that need consistent spacing
- Add custom classes for specific styling needs
- Include ids for navigation purposes
- Nest other components inside

### Don'ts
- Don't override the padding classes unless absolutely necessary
- Avoid empty sections without content
- Don't use for small UI components that don't need standard spacing

## Accessibility

The Section component uses the semantic HTML `<section>` element, which helps screen readers understand the document structure. When using the `id` prop, ensure it's unique across the page for proper navigation.
